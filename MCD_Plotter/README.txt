This is a guide for the newest version of the MCD plotter script with and without the use of mcd-b-molcas and associated scripts.

The MCD B term data can be calculated using the scripts in this directory or by using the dipole data from a gaussian tddft job
and mcd-b molcas/plot-mcdspectrum found at "https://github.com/jautschbach/mcd-molcas". 

To begin a gaussian tddft job must be ran using the following keyword: IOp(9/40=5) to increase the print threshold in the output.This is recommended in the Multiwfn manual. A sample input slurm script is provided in the Gaussian_Scripts directory.
NOTE: This significantly increases the size of the output generated by Gaussian.

Once the TDDFT job is completed you need to extract the electronic/magnetic dipole data using Multiwfn.

If you are going to use the program in the ccr you will need to use the no GUI download on their website! 
(This is recommended to save you time for larger jobs)

There is a sample script to call Multiwfn called Pyraz_mwfn.slurm with two inputs elec_mwfn.in and mag_mwfn.in which call for the creation of respective dipole files. 
NOTE: Multiwfn by default only produces files named transdipmom.txt!! This script copies them to separate files so nothing gets overwritten but be careful to run thesejobs in separate directories in case you're running them in a batch. 

NOTE: To run Multiwfn and extract the dipole data you need BOTH the .fchk file and the .out file so you will need to run the sample job in order to proceed. If you are a first time Gaussian user you produce fchk files like this:

module load gaussian/16.C.02-AVX2
formchk filename.chk (optional fchk name)

Once you have the transdipmom files you can choose which array script you would like to use depending on how you are intending to broaden the spectra.

MOLCAS METHOD
In Molcas_Scripts there is an array script in this directory alongside the example 7 output files this script should produce.
NOTE: This script only functions on files called transdipmom_elec.txt or transdipmom_mag.txt so be sure to rename them before use.
NOTE: Due to sign convention changes caused by Multiwfn magnetic dipole values are inverted with the array script.    
Be sure to check you've inverted the sign if you are using any raw data from the original transdipmom output!!!

Once you have created the necessary input files you can use the mcd-b-molcas script to calculate the B terms. Use a 1710 cm broadening factor to match a 0.15 eV broadening for spec-gen. 
NOTE: The intensity will be identical between methods. The final units are in delta ellipticity which is the experimental unit for MCD.

Once you run ./mcd-b-molcas and have your mcd-b-spectrum-# files use the mcd-b-spectrum-0 files edited to match the ideal broadening values required. The options.dat file used for the same method can be found in the Molcas_Scripts directory. 
NOTE: The states number must match identically the states used in the original gaussian calculation +1 as Molcas counts the ground state as 1 instead of 0. The same convention is kept if you are using any of the SOS keywords. (Ex: 10 states is states_sos = 11) The ! symbol is used to comment out keywords you don't want to use in the calculation.
NOTE: If you would like to use spec-gen_MCD to broaden your output data you should use the keyword specgen=.T. in the options.dat so that the output is in the 
correct format. 

Ater you have your mcd-b-spectrum files you can run ./plot-mcdspectrum < filename which will produce a graph.dat and impulses.dat file. 
NOTE: Energies in the graph.dat file are in cm^-1 so you must convert them prior to comparison to Gaussian method values.   

Gaussian Method:
Similarly to the Molcas method you need to use the array_script_gauss.py script in the Gaussian_Scripts directory to produce mu_mag/elec_array files from the transdipmom outputs.
Then you need to load the following modules in the following order:

module load gcc/11.2.0 openmpi/4.1.1
module load matplotlib/3.5.2

Once these modules are loaded you can run the MCD_plotter.py script found in the Gaussian_Scripts directory. This is run with the following inputs:

python3 MCD_plotter.py #

where the # is the maximum state you want included in the SOS approach.
NOTE: This will not crash if you ask for more states than within the file, but will only account for the maximum available states.

This will produce two files B_terms.txt and B_contributions.txt

B_terms.txt is an input file for spec-gen_MCD.py and is used for plotting the spectrum itself. It follows the same input structure as previous iterations of spec-gen with the #State# (broadening type) format. The MCD_plotter script will automatically put the number of states used at the top you need only specify mcdb or mcda for plotting.
NOTE: While spec-gen_MCD has the ability to graph mcda data the MCD_plotter script can only produce mcdb terms at present so you will need to input the data from another program
to use this functionality.

To call spec-gen the basic functionality looks like this:

python3 spec-gen_MCD.py -i input.spec -g 0.15 

where -g specifies gaussian broadening with the value of interest.
NOTE: The MCD version of spec-gen is only currently optimized for gaussian broadening, while it may work for Lorentzian, it is not recommended. 

B_contributions.txt is a file containing the MCD B term 1 and 2 from the Piepho and Schatz equation for each individual state and their combined values. This allows you to know which states contributed most to each B term. An example file can be seen in Gaussian_Scripts for the output of the MCD_plotter script.
NOTE: Pyrazine, the example has no low lying high intensity states so the first few excitations are low intensity in the Contributions/Terms output so don't be worried if your outputs contain small numbers.
